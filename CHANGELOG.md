## [Unreleased]

### Changed
- **Consolidated all runtime data under one root.** Logs and crashes were written to `~/AcoustiCAD/` while settings, projects, and backups lived in `~/Library/Application Support/AcoustiCAD/` (the "two distinct dirs" quirk). Both `main.py` and `core/speaker_profiles.py` now resolve the runtime root via Storage's `_platform_app_dir()`, so there's a single source of truth and nothing lands in the bare home folder. Removed the now-dead home-dir legacy migration (Storage already handles the `AudioSystemDesigner` rename) and the unused `LEGACY_APP_NAME` import. Reports remain in `~/Documents/AcoustiCAD Reports/` (user-visible output).

### Removed
- Retired the maintained `/Applications/AcoustiCAD.app` local install â€” the project is build-from-source only now (CI still builds `.dmg`/`.exe` on `v*` tags for anyone who wants a binary).

## [0.9.5-beta] - 2026-05-27

### Changed
- **Migrated GUI toolkit from PyQt5 to PySide6.** Same Qt underneath (Qt6 vs Qt5), but PySide6 is Qt's official LGPL Python binding â€” opens the door to non-GPL commercial licensing if AcoustiCAD ever ships paid, and gets us off PyQt5 (Qt5 LTS ended free support in 2025). Bulk mechanical substitutions across 67 `.py` files: `PyQt5â†’PySide6`, `pyqtSignalâ†’Signal`, `pyqtSlotâ†’Slot`, `pyqtPropertyâ†’Property`, `.exec_()â†’.exec()`. Hand-fixed API deltas Qt6 introduced: (a) `QAction` / `QShortcut` moved from `QtWidgets` to `QtGui` (5 files) â€” both are Qt-wide concepts, not widget-specific; (b) `QPainter.drawPolygon(*points)` no longer accepts star-unpacked args â€” replaced with `drawPolygon(points)` passing the list directly (4 sites in `zone_renderer.py`, `zones_renderer.py`, `speaker_renderer.py`). PyInstaller `.spec` hidden imports updated: `PyQt5.sip â†’ shiboken6`, `PyQt5.QtPrintSupport â†’ PySide6.QtPrintSupport`. CI Windows job switched from hardcoded `pip install "PyQt5>=..."` to `pip install -r requirements.txt` â€” single source of truth for runtime deps, no more drift risk between CI and local. `requirements-dev.txt` dropped `PyQt5-stubs` (PySide6 ships its own type stubs). Dev-mode click-through verified clean on macOS ARM; bundled `.app` test + cross-platform CI build deferred to next session.
- Renamed user reports directory from `~/Documents/AudioSystem Reports/` to `~/Documents/AcoustiCAD Reports/` â€” the last leftover string from the `AudioSystemDesigner â†’ AcoustiCAD` rename. `utils/storage.py` got a one-shot idempotent migration block (same shape as the settings-dir migration) that renames the legacy dir on first launch when the new dir doesn't exist; new dir's existence is the sentinel, no-op thereafter. `utils/report_generator.py`'s fallback path also updated to the new name.
- Downgraded `"Project {id} has no zones_data key"` from `warning` to `debug` in `ui/zones/zones_tab.py`. A new or never-edited project legitimately has no zones â€” that's normal state, not a warning. The line stays useful for debug-level tracing.

### Security
- Stripped personal identity (author/email) from `__version__.py`, `setup.py`, and `ui/dialogs/startup_dialog.py` â€” now Free Forge Labs / `support@freeforgelabs.com`. Prevents PII leaking into PyPI metadata, the About dialog, or any future public release.
- Trust-boundary validation on project file imports: `on_open_project()` now validates shape, required `name` field, `id` type, and string/list/field-count bounds before persisting. Rejects bad files with a clear error instead of letting malformed JSON poison storage. New `_validate_imported_project()` helper.

### Changed
- Refactored `ui/projects/project_details_widget.py` from hand-written field construction to a data-driven `SECTIONS` schema. Four repetitive sections (Project Details, Client & Contact, System Requirements, Project Information â€” 14 fields total) now live in a `list[FieldSpec]` table with `kind="text" | "phone" | "chips"` discriminator; `_build_schema_section()` and `load_project_data()` loop over it instead of duplicating per-field plumbing in two places. Header, description, power, and notes blocks stay hand-built â€” they're one-of-a-kind shapes. Adding a new field is now one `SECTIONS` entry instead of three places (declare, add `FieldRow`, set text in load). Public surface unchanged (`load_project_data`, `clear_project_details`). Smoke-tested and verified live.
- Log rotation in `error_handler.setup_logging()` now uses stdlib `logging.handlers.TimedRotatingFileHandler` (daily rotation at midnight, `backupCount=7`) instead of the prior glob-and-unlink cleanup pass. Active log file flipped from `app_YYYYMMDD.log` to `app.log`; rotated files become `app.log.YYYY-MM-DD`. Cleaner, no brittleness around clock changes or unusual filenames. Note: existing `app_YYYYMMDD.log` files in `~/AcoustiCAD/logs/` won't be touched by the new handler â€” sweep manually if you want them gone.
- All `logging.getLogger("hardcoded_name")` calls across ~43 files swept to `logging.getLogger(__name__)`. Log output now shows fully-qualified module paths (e.g. `ui.plotter.rendering.zone_renderer`) instead of bare short names â€” easier to trace where a log line came from. `ui/zones/properties_panel.py` also gained a module-level `logger = logging.getLogger(__name__)` to replace three inline `logging.getLogger(...).error(...)` calls at error sites.
- Project IDs now generated with `uuid.uuid4().hex` instead of `int(time.time())`. Wall-clock IDs collide on rapid creation (loops, scripts, same-second clicks) and silently overwrite existing projects on save. Existing projects keep their int IDs; validator accepts both types. **Session 3 follow-up:** moved the fix from `project_tab.py`'s dead `else:` branch to `core/project_manager.py:create_new_project` (the actually-executing path). The original session 2 fix had landed in unreachable code; the real prod path was still using wall-clock seconds.
- Collapsed dual-path `on_new_project()` in `ui/projects/project_tab.py` â€” `project_manager` is now a required constructor argument (was `=None`-default), the dead `else:` branch is removed, and two more dead `if self.project_manager:` defensive guards eliminated. `ProjectTab` fails loud at construction if `project_manager` is missing instead of silently taking a divergent code path.
- Replaced custom `find_main_window()` parent-walker in `project_tab.py` with Qt-native `self.window()`. Custom walker had scar-tissue guard code (`if callable(getattr(parent, 'parent', None))`) reinventing what Qt's C++ implementation already does. Defensive `hasattr(top, 'load_project')` check preserved so the existing fallback path still fires if `ProjectTab` is ever embedded outside `MainWindow`.
- Moved the inline global QApplication stylesheet from `main.py` to `BaseStyles.get_global_app_stylesheet()` in `ui/styles/base_styles.py`. Pure refactor â€” verbatim QSS move, no visual change. `ui/styles/` is now the single home for all QSS in the project. Token-ifying the hardcoded colors to design tokens is deferred until we want a visible style refresh.
- All project file I/O in `on_open_project()` / `on_export_project()` now declares `encoding='utf-8'`. Without it, Python on Windows defaults to `cp1252` and crashes on non-ASCII (Ă©, em-dash, smart quotes, emoji) in projects saved on Mac. Export also adds `ensure_ascii=False` so JSON files write actual UTF-8 instead of `\uXXXX` escape sequences.
- `ConfigManager` now takes separate `config_dir` (bundled defaults, read-only when installed) and `user_config_dir` (writable user overrides). User config now lives in `~/Library/Application Support/AcoustiCAD/config/` matching Storage's OS-standard data location. Was silently broken for packaged `.app` builds since `/Applications/AcoustiCAD.app/` is read-only.
- Crash reports (`error_handler.py`) now include the AcoustiCAD version and write with `encoding='utf-8'`. Crash dialog points users to `support@freeforgelabs.com` for triage instead of "please consider reporting" with no channel.
- `ProjectItemWidget` no longer defines a custom `clicked` signal or overrides `mousePressEvent`. The parent QListWidget's native click handling drives selection â€” the custom signal was duplicating that work via a lambda round-trip.
- `main.py` removed the dead `on_last_window_closed` callback. The real unsaved-changes prompt lives in `MainWindow.closeEvent`, which fires *before* the close decision is made (the lastWindowClosed hook fires after, so it could only log, not prompt).
- Completed `AudioSystemDesigner â†’ AcoustiCAD` rename across `APP_ID`, settings dir, runtime dir, PyInstaller spec filenames, CI workflow, `setup.py`, `Readme.md`, and `installer/setup.iss`.
- Consolidated version to a single source of truth: the `VERSION` file at project root. `__version__.py` reads it at import (with `sys._MEIPASS` fallback for PyInstaller-frozen runs), `Storage.VERSION` derives from `__version__`, both `.spec` files compute `BUNDLE_VERSION` from `VERSION` at build time, CI workflow uses `$(cat VERSION)` for DMG names, `installer/setup.iss` reads `VERSION` via Inno preprocessor.
